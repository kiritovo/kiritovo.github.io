Posted on 2020-05-15

java项目http变为https

  1. 本次的项目是tomcat7，ssh项目的。我们需要修改的文件

    tomcat 的server.xml 和项目的web.xml

    首先要确定8443端口没有被占用

    使用管理员打开cmd的黑框 这里我们密码用123456

    执行 keytool -genkey -alias tomcat -keyalg

    是否正确输入y 然后按回车就ok了

    这样在C:/user/用户名 目录下会生成一个.keystore结尾的文件

  2. 配置tomcat使用 keystore文件

    打开 config/server.xml 找到下面这段并被注释的 一般未修改就是已经注释的

    新增以下内容

    这里的 file 路径keystoreFile根据自己的计算机用户名进行修改 密码keystorePass也是

  3. 测试：启动tomcat，然后访问https://localhost:8443 你会看到tomcat的初始页，访问8080端口还是有效的

  4. 配置项目使用SSL：找到项目的web.xml文件 增加如下配置

    将 URL 映射设为 /* ，这样你的整个应用都要求是 HTTPS 访问，而 transport-guarantee 标签设置为 CONFIDENTIAL 以便使应用支持 SSL。

    如果你希望关闭 SSL ，只需要将 CONFIDENTIAL 改为 NONE 即可。

    向 CA 提交域名及公司信息申请数字证书就可以了。当然了，数字证书的申请需要花费年费的，一年 4000 到 20000 元不等。
    虽然 JDK 的 keytool 工具也可以免费制作自签名的证书，但这只能用在练习或者测试中，因为如果数字证书颁布商不在浏览器的信任列表中，是会给用户弹出警告框的。作为电子商务网站肯定要用商业的数字证书！
